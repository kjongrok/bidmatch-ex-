# BidMatch 설계 문서

나라장터 공공 API 데이터를 수집하고, 사용자 조건과 일치하는 공고를 이메일로 알림 전송하는 웹 애플리케이션의 설계 문서 저장소입니다.

## 문서 구조

| 폴더 | 목적 |
| --- | --- |
| `01-requirements` | 서비스 요구사항, 기능 명세, 사용자 시나리오 정리 |
| `02-architecture` | 전체 시스템 구조, 백엔드/프론트엔드 구성, 배치/스케줄링 흐름 |
| `03-database` | ERD, 테이블 정의서, 데이터 보관/정합성 정책 |
| `04-api` | 내부 API 명세, 외부 공공 API 연동 명세, 요청/응답 예시 |
| `05-design` | 화면 설계, 사용자 흐름, 디자인 가이드 |
| `06-email` | 이메일 알림 정책, 템플릿, 발송 실패/재시도 정책 |
| `07-operations` | 배포, 환경 변수, 모니터링, 장애 대응, 보안 체크리스트 |
| `08-decisions` | 주요 기술/정책 결정 기록 |
| `09-troubleshooting` | Notion 기반 트러블슈팅 기록 운영 구조, 속성, 템플릿 |
| `assets` | 이미지, 다이어그램, 와이어프레임 등 문서용 리소스 |

## 작성 규칙

- 문서는 Markdown(`.md`)을 기본으로 작성합니다.
- 다이어그램은 Mermaid 또는 이미지 파일로 관리합니다.
- 확정되지 않은 내용은 `TBD`로 표시하고, 결정된 내용은 `08-decisions`에 기록합니다.
- 장애, 오류, 해결 과정은 Notion의 `Trouble Logs`에 기록하고, 프로젝트 구분은 `Project` 속성으로 관리합니다.
