# Components

여러 화면에서 재사용하는 UI 컴포넌트를 보관합니다.

## 권장 하위 분류

- `layout`: 사이드바, 헤더, 앱 레이아웃
- `common`: 버튼, 배지, 카드, 입력 필드
- `notice`: 공고 목록/상세 관련 컴포넌트
- `rule`: 관심 조건 관련 컴포넌트
- `admin`: 관리자 화면 관련 컴포넌트

## 작성 기준

- 컴포넌트는 가능한 한 props로 데이터를 받도록 작성합니다.
- API 호출은 컴포넌트 내부에 직접 넣지 않고 `services` 또는 `hooks`로 분리합니다.
- 사용자 입력을 HTML 문자열로 직접 렌더링하지 않습니다.
